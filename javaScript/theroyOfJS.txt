        Varibles in js
------------------------------
 Three ways of create variable:
    1.var
    2.let
    3.const

    1. var:
        - It is function-scoped, meaning it is accessible within the function it is declared in.
        - It can be re-declared and updated within its scope.
        - It is hoisted to the top of its scope, meaning it can be used before it is declared.
    2. let:
        - It is block-scoped, meaning it is only accessible within the block it is declared in.
        - It can be updated but not re-declared within its scope.
        - It is not hoisted, meaning it cannot be used before it is declared.
    3.const:
        - It is block-scoped, similar to let.
        - It cannot be updated or re-declared within its scope.
        - It must be initialized at the time of declaration.

                Template Literal in Detail
------------------------------------------------------------------------------------------------------------
    Template literals are a feature in JavaScript that allows for easier string interpolation and multi-line strings. They are enclosed in backticks (`) instead of single or double quotes. Template literals provide a more convenient way to create strings that include variables and expressions.

    1. String Interpolation:
        Template literals allow you to embed expressions inside a string using the `${}` syntax. This makes it easier to create dynamic strings without the need for concatenation.
        Example:
            const name = "Alice";
            const age = 30;
            const message = `My name is ${name} and I am ${age} years old.`;
            console.log(message); // Output: My name is Alice and I am 30 years old.

    2. Multi-line Strings:
        Template literals also support multi-line strings without the need for escape characters. You can simply press Enter to create a new line within the string.
        Example:
            const multiLineString = `This is a multi-line string.
            It can span multiple lines without any issues.
            This makes it easier to read and maintain.`;
            console.log(multiLineString);
            /* Output:
                This is a multi-line string.
                It can span multiple lines without any issues.
                This makes it easier to read and maintain.
            */

    3. Expression Evaluation:
        You can include any valid JavaScript expression inside the `${}` syntax, and it will be evaluated and included in the resulting string.
        Example:
            const a = 5;
            const b = 10;
            const result = `The sum of ${a} and ${b} is ${a + b}.`;
            console.log(result); // Output: The sum of 5 and 10 is 15.

    In summary, template literals provide a more convenient and readable way to create strings that include variables, expressions, and multi-line content in JavaScript.

                        Operators in js
    -----------------------------------------------------------------------------------------------------------
    1. Arithmetic Operators: +, -, *, /, %, ++, --
    2. Assignment Operators: =, +=, -=, *=, /=, %=
    3. Comparison Operators: ==, ===, !=, !==, >, <, >=, <=
    4. Logical Operators: &&, ||, !
    5. Bitwise Operators: &, |, ^, ~, <<, >>, >>>
    6. Ternary Operator: condition ? expression1 : expression2
    7. Type Operators: typeof, instanceof

    1.Arithmetic Operators:
    - + (Addition): Adds two operands together.
    - - (Subtraction): Subtracts the second operand from the first.
    - * (Multiplication): Multiplies two operands together.
    - / (Division): Divides the first operand by the second.
    - % (Modulus): Returns the remainder of the division of the first operand by the second.
    - ++ (Increment): Increases the value of the operand by 1.
    - -- (Decrement): Decreases the value of the operand by 1.

    2.Assignment Operators:
    - = (Assignment): Assigns the value of the right operand to the left operand.
    - += (Addition Assignment): Adds the right operand to the left operand and assigns the result to the left operand.
    - -= (Subtraction Assignment): Subtracts the right operand from the left operand and assigns the result to the left operand.
    - *= (Multiplication Assignment): Multiplies the left operand by the right operand and assigns the result to the left operand.
    - /= (Division Assignment): Divides the left operand by the right operand and assigns the result to the left operand.
    - %= (Modulus Assignment): Takes the modulus of the left operand by the right operand and assigns the result to the left operand.

    3.Comparision Operation:
    - == (Equality): Compares two operands for equality after type coercion.
    - === (Strict Equality): Compares two operands for equality without type coercion.
    - != (Inequality): Compares two operands for inequality after type coercion.
    - !== (Strict Inequality): Compares two operands for inequality without type coercion.
    - > (Greater Than): Checks if the left operand is greater than the right operand.
    - < (Less Than): Checks if the left operand is less than the right operand.
    - >= (Greater Than or Equal To): Checks if the left operand is greater than or equal to the right operand.
    - <= (Less Than or Equal To): Checks if the left operand is less than or equal to the right operand.

    4.Logical Operators:
    - && (Logical AND): Returns true if both operands are true, otherwise returns false.    
    - || (Logical OR): Returns true if at least one of the operands is true, otherwise returns false.
    - ! (Logical NOT): Returns true if the operand is false, and false if the operand is true.

    5.Bitwise Operators:
    - & (Bitwise AND): Performs a bitwise AND operation on two operands.
    - | (Bitwise OR): Performs a bitwise OR operation on two operands.
    - ^ (Bitwise XOR): Performs a bitwise exclusive OR operation on two operands.
    - ~ (Bitwise NOT): Performs a bitwise NOT operation on a single operand.
    - << (Left Shift): Shifts the bits of the first operand to the left by the number of positions specified by the second operand.
    - >> (Right Shift): Shifts the bits of the first operand to the right by the number of positions specified by the second operand.
    - >>> (Unsigned Right Shift): Shifts the bits of the first operand to the right by the number of positions specified by the second operand, filling the leftmost bits with zeros.

    6.Ternary Operator:
    - condition ? expression1 : expression2: Evaluates the condition and returns expression1 if the condition is true, or expression2 if the condition is false.
    
    7.Type Operators:
    - typeof: Returns a string indicating the type of the operand.
    - instanceof: Tests whether an object is an instance of a specific constructor or class.

            conditional Statements in javaScripts
------------------------------------------------------------
        1. if Statement
        2. if...else Statement
        3. if...else if...else Statement
        4. switch Statement
        Note: to input from the user the function called prompt();
        1.if statement:
        ***********************
        The if statement is used to execute a block of code if a specified condition is true.
        Syntax:
            if (condition) {
                // code to be executed if the condition is true
            }
        Example:
            let age = 18;
            if (age >= 18) {
                console.log("You are an adult.");
            }
        2.if...else statement:
        *************************
        The if...else statement is used to execute one block of code if a specified condition is true, and another block of code if the condition is false.
        Syntax:
            if (condition) {
                // code to be executed if the condition is true
            } else {
                // code to be executed if the condition is false
            }
        Example:
            let age = 16;
            if (age >= 18) {
                console.log("You are an adult.");
            } else {
                console.log("You are a minor.");
            }
        3.if...else if...else statement:
        ***********************************
        The if...else if...else statement is used to execute different blocks of code based on multiple conditions.
        Syntax:
            if (condition1) {
                // code to be executed if condition1 is true
            } else if (condition2) {
                // code to be executed if condition2 is true
            } else {
                // code to be executed if both conditions are false
            }
        Example:
            let score = 85;
            if (score >= 90) {
                console.log("Grade: A");
            } else if (score >= 80) {
                console.log("Grade: B");
            } else if (score >= 70) {
                console.log("Grade: C");
            } else {
                console.log("Grade: F");
            }
        4.switch statement:
        ***************************
        The switch statement is used to perform different actions based on different conditions. It is often used as an alternative to multiple if...else if statements when there are many conditions to evaluate.
        Syntax:
            switch (expression) {
                case value1:
                    // code to be executed if expression === value1
                    break;
                case value2:
                    // code to be executed if expression === value2
                    break;
                // more cases...
                default:
                    // code to be executed if expression doesn't match any case
            }
        Example:
            let day = "Monday";
            switch (day) {
                case "Monday":
                    console.log("Today is Monday.");
                    break;
                case "Tuesday":
                    console.log("Today is Tuesday.");
                    break;
                case "Wednesday":
                    console.log("Today is Wednesday.");
                    break;
                default:
                    console.log("It's another day.");
            }

            Type conversion in javaScript  
-----------------------------------------------------------------
Type conversion in JavaScript refers to the process of converting a value from one data type to another. 
This can happen implicitly (automatic type conversion) or explicitly (manual type conversion).
1. Implicit Type Conversion:
   JavaScript automatically converts values to the appropriate type when performing operations that involve different data types. 
   For example:
        let result = "5" + 10; // result will be "510" (string concatenation)
        let sum = "5" - 2; // sum will be 3 (string is converted to number)
2. Explicit Type Conversion:
    You can manually convert values to a specific type using built-in functions or methods. 
    For example:
          let numString = "123";
          let num = Number(numString); // num will be 123 (string converted to number)
          let boolValue = Boolean(0); // boolValue will be false (0 is falsy)
          let strValue = String(true); // strValue will be "true" (boolean converted to string)

    Syntax:
          do {
                // code to be executed
          } while (condition);
    Example:
          let i = 0;
          do {
                console.log(i);
                i++;
          } while (i < 5);

                            Function in javaScript
--------------------------------------------------------------------------------------------
Functions in JavaScript are reusable blocks of code that perform a specific task.
They can be defined using function declarations or function expressions.
1. Function Declaration:
   A function declaration defines a named function that can be called before it is defined in the code.
   Syntax:
        function functionName(parameters) {
            // code to be executed
        }
   Example:
        function greet(name) {
            return "Hello, " + name + "!";
        }
        console.log(greet("Alice")); // Output: Hello, Alice!
2. Function Expression:
    A function expression defines a function as part of a larger expression, such as an assignment or an argument to another function.
    Syntax:
          const functionName = function(parameters) {
                // code to be executed
          };
    Example:
          const greet = function(name) {
                return "Hello, " + name + "!";
          };
          console.log(greet("Bob")); // Output: Hello, Bob!
3. Arrow Function:
    Arrow functions provide a shorter syntax for writing function expressions. 
    They do not have their own 'this' context and are often used for simple, concise functions.
    Syntax:
          const functionName = (parameters) => {
                // code to be executed
          };
    Example:
          const greet = (name) => {
                return "Hello, " + name + "!";
          };
          console.log(greet("Charlie")); // Output: Hello, Charlie!

4. Named functions
    Named functions are function expressions that have a name. 
    They can be useful for debugging and recursion.
    Example:
          const factorial = function fact(n) {
                if (n === 0) {
                      return 1;
                }
                return n * fact(n - 1);

5. Anonymous function
^^^^^^^^^^^^^^^^^^^^^^^
    Anonymous functions are function expressions that do not have a name. 
    They are often used as arguments to other functions or for immediate execution.
    Example:
          setTimeout(function() {

6. immediately Invoked function expression(IIFE)
    An immediately invoked function expression (IIFE) is a function that is defined and executed immediately after its creation. 
    It is often used to create a new scope and avoid polluting the global namespace.
    Syntax:
          (function() {
                // code to be executed
          })();
    Example:
          (function() {
                console.log("This is an IIFE.");
          })();

--------------------------------------------------------------------
                Scope in JavaScript
--------------------------------------------------------------------
1. globe level:
    Variables declared outside of any function or block are in the global scope and can be accessed from anywhere in the code.
    Example:
          var globalVar = "I am a global variable.";
          console.log(globalVar); // Output: I am a global variable.
2. function level:
    Variables declared within a function are in the function scope and can only be accessed within that function.
    Example:
          function myFunction() {
                var localVar = "I am a local variable.";
                console.log(localVar); // Output: I am a local variable.
          }
          myFunction();
          console.log(localVar); // Error: localVar is not defined
3. block level:
    Variables declared with let or const within a block (e.g., inside an if statement or a loop) are in the block scope and can only be accessed within that block.
    Example:
          if (true) {
                let blockVar = "I am a block variable.";
                console.log(blockVar); // Output: I am a block variable.
          }
          console.log(blockVar); // Error: blockVar is not defined

                        Default and Rest Parameters in java script
----------------------------------------------------------------------------------------------------
1. Default Parameters:
   Default parameters allow you to specify default values for function parameters. If a parameter is not provided when the function is called, the default value will be used.
   Syntax:
        function functionName(parameter1 = defaultValue1, parameter2 = defaultValue2) {
            // code to be executed
        }
   Example:
        function greet(name = "Guest") {
            return "Hello, " + name + "!";
        }
        console.log(greet()); // Output: Hello, Guest!
        console.log(greet("Alice")); // Output: Hello, Alice!
2. Rest Parameters:
    Rest parameters allow you to represent an indefinite number of arguments as an array. 
    They are denoted by three dots (...) followed by the parameter name.
    Syntax:
          function functionName(...restParameter) {
                // code to be executed
          }
    Example:
          function sum(...numbers) {
                return numbers.reduce((total, num) => total + num, 0);
          }
          console.log(sum(1, 2, 3)); // Output: 6
          console.log(sum(4, 5)); // Output: 9          
Note:arguments is object in the every function that store the all parameters

            console.log(arguments[0]) // gives the first parameter

        }

------------------------------------------------------------------------------------------------------
                        forEach Method in the Detail
------------------------------------------------------------------------------------------------------
The forEach method is an array method in JavaScript that allows you to iterate over each element in an array 
and execute a provided function for each element.
Syntax:
    array.forEach(function(currentValue, index, array) {
        // code to be executed for each element
    });
Parameters:
- currentValue: The current element being processed in the array.
- index (optional): The index of the current element being processed.
- array (optional): The array that forEach is being called on.
Example:
    const numbers = [1, 2, 3, 4, 5];
    numbers.forEach(function(num) {
        console.log(num);
    });
    // Output:
    // 1
    // 2
    // 3
    // 4
    // 5
The forEach method does not return a new array;
it simply executes the provided function for each element in the original array.

    The map() method is an array method in JavaScript that creates a new array by applying a provided function 
    to each element of the original array. 
    It does not modify the original array.

                                reduce in javaScript
-----------------------------------------------------------------------------------------------------------------------

    The reduce() method is an array method in JavaScript that executes a reducer function on each element of the array, 
    resulting in a single output value. 
    It does not modify the original array.
    Syntax:
        let result = originalArray.reduce(function(accumulator, currentValue, index, array) {
            // code to be executed for each element
            // return the updated accumulator
        }, initialValue);
    Example:

        const sum = numbers.reduce(function(total, num) {
            return total + num; // Accumulate the sum of the numbers
        }, 0); // Initial value of the accumulator is 0

    Note: The reduce() method can also be used with arrow functions for a more concise syntax.
    Example with arrow function:
        const numbers = [1, 2, 3, 4, 5];
        const sum = numbers.reduce((total, num) => total + num, 0);
        console.log(sum); // Output: 15

----------------------------------------------------------------------------------------------------------------
                             call, apply,bind in javaScript
----------------------------------------------------------------------------------------------------------------
==>call, apply, and bind are methods in JavaScript that allow you to change the context (the value of this) of a function when it is called.
1. call():
    The call() method calls a function with a given this value and arguments provided individually.
    Syntax:
        functionName.call(thisArg, arg1, arg2, ...);
    Example:
        function greet() {
            console.log("Hello, " + this.name + "!");
        }
        const person = { name: "Alice" };
        greet.call(person); // Output: Hello, Alice!

2. apply():
    The apply() method calls a function with a given this value and arguments provided as an array (or an array-like object).
    Syntax:
        functionName.apply(thisArg, [argsArray]);
    Example:
        function greet(greeting) {
            console.log(greeting + ", " + this.name + "!");
        }
        const person = { name: "Bob" };
        greet.apply(person, ["Hi"]); // Output: Hi, Bob!
3. bind():
    The bind() method creates a new function that, when called, has its this keyword set to the provided value, with a given sequence of arguments preceding any provided when the new function is called.
    Syntax:
        let boundFunction = functionName.bind(thisArg, arg1, arg2, ...);
    Example:
        function greet() {
            console.log("Hello, " + this.name + "!");
        }
        const person = { name: "Charlie" };
        const boundGreet = greet.bind(person);
        boundGreet(); // Output: Hello, Charlie!
